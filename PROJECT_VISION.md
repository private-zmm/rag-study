# 跨端知识库笔记项目总览

## 1. 项目目标

本项目计划做成一款跨端知识库与笔记应用，兼容主流平台，既可以作为个人知识管理工具，也可以作为面向大模型的私有知识库入口。

本项目不采用本地优先模式。笔记、知识库、对话、系统配置等核心数据统一保存在服务端相关数据库中，客户端主要负责展示、编辑和交互。

目标支持的平台包括：

- Windows
- macOS
- iOS
- Android
- Web
- Docker 私有化部署

## 2. 核心定位

应用的核心是「知识库 + 笔记 + 大模型对话」。

用户可以在线编辑 Markdown 笔记、组织知识、剪藏网页内容，并将这些内容默认向量化，自动沉淀为可检索、可问答的知识库。

## 4. 关键技术决策

- 前端使用 React + TypeScript + Vite
- 后端使用 Java + Spring Boot
- 主数据库使用 PostgreSQL
- 向量数据库使用 Qdrant
- 文件和备份支持 S3 兼容对象存储
- 首页参考 open-webui，默认进入大模型对话
- Markdown 笔记支持在线编辑
- 网页剪藏通过应用内 URL 抓取实现
- 笔记、知识库、对话和配置不存本地文件，统一保存到服务端相关数据库

## 7. 学习式开发原则

本项目采用边学习边开发的方式推进。

每个功能模块都按照以下节奏进行：

- 先解释目标：这个模块要解决什么问题
- 再解释设计：为什么这样拆分目录、接口、数据库表和服务
- 然后开始实现：每次只实现一个清晰的小目标
- 实现后运行验证：通过启动项目、调用接口、查看页面或执行测试确认结果
- 最后做小结：总结本次学到的概念、代码结构和后续可以扩展的方向

开发过程中优先保证：

- 每一步都能理解
- 每一步都能运行
- 每一步都能回头复盘
- 不一次性堆太多代码
- 不为了赶进度跳过基础概念

协作方式上，Codex 需要以带教方式参与开发：

- 写代码前说明为什么这样做
- 写代码后解释关键文件和关键代码
- 遇到错误时带着分析原因，而不是只给最终答案
- 对重要技术点补充必要的背景知识
- 帮助形成可持续的项目开发习惯
