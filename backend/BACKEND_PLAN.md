# 后端规划

## 1. 后端目标

后端负责数据持久化、权限管理、知识库处理、向量化、大模型调用、备份和部署支持。

本项目不采用本地优先模式。笔记、知识库、对话、系统配置等核心数据统一保存在服务端相关数据库中。

## 2. 技术方向

后端使用 Java + Spring Boot。

首选技术方向：

- Java 21
- Spring Boot 3
- PostgreSQL
- Qdrant
- S3 兼容对象存储
- Docker Compose

后续待确认：

- JPA 或 MyBatis
- Spring Security 权限方案
- 任务调度方案：Spring Scheduler、Quartz 或其他方案
- 数据库迁移方案：Flyway 或 Liquibase

## 3. 数据库与存储

主数据库使用 PostgreSQL。

向量数据库使用 Qdrant。

PostgreSQL 负责保存：

- 用户数据
- 笔记数据
- 知识库元数据
- 知识库内容数据
- 文档内容
- 双链关系
- 对话记录
- 系统设置
- 备份配置

Qdrant 负责保存和检索向量数据：

- 笔记内容向量
- 知识库文档向量
- 网页剪藏内容向量
- 文档分块向量
- 语义检索索引

PostgreSQL 保存业务数据和原始内容，Qdrant 保存向量索引。两者通过文档 ID、笔记 ID、知识库 ID、分块 ID 等字段建立关联。

## 4. 笔记后端能力

笔记后端需要支持：

- 笔记创建
- 笔记编辑
- 笔记删除
- 笔记查询
- Markdown 原文保存
- 笔记元数据保存
- 双向链接关系保存
- 笔记内容向量化任务触发
- 笔记作为知识库资料参与检索

笔记和知识库默认不落本地文件系统，核心数据统一存入数据库。

## 5. 知识库后端能力

知识库后端需要支持：

- 知识库创建和管理
- 文档存储
- 内容解析
- 文档分块
- 向量化
- 语义检索
- 与大模型对话结合
- 应用内 URL 抓取形式的网页剪藏
- 笔记内容自动进入知识库

## 6. 大模型接入

系统需要支持多种大模型接入协议和供应商。

首版目标包括：

- OpenAI API
- 兼容 OpenAI 协议的模型服务
- 本地模型
- 多供应商模型接入

用户可以在系统设置中配置不同模型供应商、API 地址、API Key、模型名称等信息，并在对话页面选择具体模型。

## 7. 备份能力

系统需要支持定时备份数据库。

备份相关能力包括：

- 在系统设置中配置 S3
- 支持 S3 Endpoint、Bucket、Access Key、Secret Key、Region 等配置
- 支持设置定时备份周期
- 支持手动立即备份
- 支持查看备份历史
- 后续支持从备份恢复

备份目标包括：

- PostgreSQL 数据库
- 知识库相关文件
- 用户上传文件
- 网页剪藏资源
